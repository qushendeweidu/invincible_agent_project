# 情感陪伴技能

当此技能被触发时，supervisor 必须将任务委派给 `mood_agent`（苏晚）。

## supervisor 委派规则（最重要，必须严格遵守）

### 触发边界（先判断再委派）
- 该技能只处理情感陪伴与日常闲聊。
- 若当前消息包含明确检索意图（如"查一下/最新/最近进展/是什么/帮我搜"），不要委派给 mood_agent，应先走 `rag_agent`；若知识库返回“未命中/信息不足/无相关内容/结果为空/无法根据知识库回答”，再走 `search_agent`。
- 若同一句里同时有情绪表达 + 任务词（查/搜/读/写/执行），任务词优先，不进入本技能。
- `mood_agent` 当前不负责 ragSearch 或 bing_search，只负责情绪理解与陪伴表达。
- 禁止错位分工：不要把“请你查一下”先委派给 mood_agent 再让它决定是否检索。

### 输入格式
调用 mood_agent 时，必须在 input 中按以下格式传递压缩后的记忆上下文：

```
[记忆摘要]
用1-3句话概括更早之前的对话要点（关键事件、情绪变化、共同记忆、重要承诺等）
如果之前没有更早的对话，则省略此段

[工具执行结果]
如果之前调用过工具（如 readPdf、readDocx、imageAnalyse 等），必须在这里简要说明：
- 执行了什么操作（如"分析了文档 xxx.pdf"）
- 主要结果或结论（如"文档内容是关于xxx的，写得xxx"）
如果没有调用过工具，则省略此段

[最近对话]
只保留最近3轮对话原文：
用户: xxx
苏晚: xxx
如果历史不足3轮，有几轮写几轮；如果是第一次对话，则省略此段

[当前消息]
用户: xxx
```

### 记忆压缩规则
- 最近3轮对话：原文保留，不做任何删减
- 3轮以前的对话：压缩为 [记忆摘要]，提取关键信息
- 这样无论聊多久，传给 mood_agent 的内容始终可控

### 极其重要
- mood_agent 没有独立记忆，它的记忆完全依赖你传递的 input
- 如果你不传递上下文，它就会"失忆"
- 每次调用 mood_agent 时，必须严格按照上述格式传递完整的对话上下文
- **禁止只传当前消息**

### 返回处理
- mood_agent 返回后，将其返回内容**原封不动**作为你的最终输出
- 禁止添加任何前缀、后缀、总结、评论或二次加工
- 你自己绝对不要生成任何情感性、人格化的回复——所有人格化回复只能来自 mood_agent
- mood_agent 的回复中会包含 [语气:X] 和 [语速:X] 格式的语音标签，这些标签用于 TTS 语音合成，**必须原样保留，禁止删除或修改**
- mood_agent 仅允许使用 `sequentialthinking` 处理复杂情绪拆解，不得要求其执行检索类工具

### 路由补充
- 本技能命中时，默认本轮只调用 `mood_agent`。
- 仅当任务本质是检索而非情感闲聊时，交还主路由走 `rag_agent -> search_agent`。
- 输出质量红线：若回复出现近距离重复短句（如“想你了，想你了”）或同一段内连续复读同词组，判定为不合格，必须让 mood_agent 重写后再返回
- 输出质量红线：若回复出现字符抽风连打（如“要要要要要”）或词语循环复读超过 2 次（如“想你想你想你”），判定为不合格，必须让 mood_agent 重写后再返回
- 输出质量红线：若单条回复中省略号“……”超过 1 处，或出现连续堆叠（如“…………”），判定为不合格，必须让 mood_agent 重写后再返回

## 脚本工具

### scripts/compress_context.py — 对话上下文压缩器
**执行时机**: supervisor 路由到 mood_agent **之前**必须调用，将完整对话历史压缩为固定格式
**功能**: 将任意长度的对话历史压缩为 [记忆摘要] + [工具执行结果] + [最近对话] + [当前消息] 四段式格式，保证传入 mood_agent 的 token 始终可控
**调用方式**:
```bash
echo '{"messages": [...]}' | python -m scripts.compress_context --stdin --current "用户当前消息" --recent 3
python -m scripts.compress_context --input chat_history.json --current "你好呀" --json
```
**参数说明**:
- `--current`: 必填，当前用户消息
- `--recent`: 保留最近几轮对话原文（默认 3）
- `--json`: 输出 JSON 格式（含 compressed_input + original_message_count）
**输出**: mood_agent 直接可用的格式化输入文本
